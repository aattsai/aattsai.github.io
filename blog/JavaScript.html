JavaScript vs. Ruby - Hashes and Objects

After 4 weeks of intensive Ruby challenges, this week we are finally diving into the world of JavaScript. Programmers I speak to often have mixed feelings about JavaScript. I often hear high praises about the language, but other times, I hear people expressing clear feeling of frustration towards it. One of the very first things I learned this week was that JavaScript, unlike its name may suggest, has essentially no connection with Java. However, after a few hours of JavaScript, I actually feel there are some resemblances between JavaScript and Java, perhaps just from a purely syntactic perspective.

Back to the topic of this blog post. This week I would like to discuss one of the key topics in the JavaScript language: objects. In JavaScript, object groups together a set of variables and functions and store that information within its umbrella. It is comparable to Hashes in Ruby. Similar to Hashes, objects can be defined by using curly braces and stored in a variable. Within the curly braces, variables are stored called properties, which are represented by keys that correspond to values. An object cannot have two keys with the same name. Interestingly, a function can also be stored as a value in an object. When a function is stored within an object, it is called a method.

There are two common ways to create an object in JavaScript. The first and easiest way is called literal notation:

var car = {
   brand: "bmw",
   doors: 4,
   horsepower: 300
};

In this example, an object variable called car is defined and stored with 3 properties. Brand, doors, and horsepower are the keys of the properties each with a corresponding value. To access properties, the easiest way is to use the dot notation. For example, if I want to know the number of doors in this object car. I can do car.doors to access the value 4 in this case.

The second way to create an object in JavaScript is by using a constructor notation:


var car = new Object();
   car.brand = "bmw";
   car.doors = 4;
   car.horsepower = 300;

In this example, I used the keyword new and the object constructor to first create an empty object called car. Then each of the properties are added separately by calling the object car and the keys with assigned value. Note that this will result in an identical object as the one above created by literal notation.

A more complicated but very useful way to create an object by constructor notation is by using a function as template:


function Car(brand, doors, horsepower) {
   this.brand = brand;
   this.doors = doors;
   this.horsepower = horsepower;
};

In this example, a function template is used to create an object that accepts three arguments, which represent the keys of each properties. The keyword this is used to indicate that the property belongs to the object that this function creates. Using a function template to create object allows one to create multiple instances of the object by calling the constructor function after the keyword new. It is especially useful when one wishes to create several objects to represent similar things.
